Date Time Picker Dialog

{% highlight xml %} {% endhighlight %}

Initializing DateTimePicker

It is important to call initialize(activity) before using the DateTimePicker as the calling activity is necessary to call the DateTimePickerDialog.
{% highlight java %} dateTimePicker.initialize(this); {% endhighlight %}

Setting DateTimeChangedListener

A callback with the selected date and time is made if a DateTimeChangedListener is set to the DateTimePicker component after initializing

{% highlight java %} dateTimePicker.setDateTimeChangedListener(new DateTimePicker.DateTimeChangedListener() { @Override public void onDateTimeSelected(String formatedDate, String formatedTime, Date selectedDate) { //retrieve selected date and time. The selectedDate Date object contains the time as well //the formattedDate and formattedTime String values are formatted as 12 August, 2018 and 10:00 AM } @Override public void onDateRemoved() { //handle date value removed event } }); {% endhighlight %} Or if the activity/fragment implements it, set the DateTimeChangedListener like this: {% highlight java %} dateTimePicker.setDateTimeChangedListener(this); {% endhighlight %}

Additionally, the start time and end time of the date picker can be set. This will be the range of the time picker.

{% highlight java %} dateTimePicker.setRangeStartTime("2018-01-1T7:30:30.042+08:00"); dateTimePicker.setRangeEndTime("2018-10-20T22:00:30.042+08:00"); {% endhighlight %}

SearchBox

SearchBox is an extension of an Interactive Panel where the user can tap on the search box button to receive a callback. Additionally, a search box button has a 'next' button implementation which has a callback of its own. The next button can be toggled as enabled and disabled to toggle visibility.

{% highlight xml %} {% endhighlight %}

Callbacks

{% highlight java %} searchBox.setListener(new SearchBox.SearchBoxClickListener() { @Override public void onNextClicked() { //handle next button clicks } @Override public void onSearchBoxClicked() { //handle search box clicks } }); {% endhighlight %}

Set Attributes from code level

Visibility of the next button and the hint can be changed from code level.

{% highlight java %} searchBox.setHint("My Custom Text"); searchBox.setDisplayNext(true); {% endhighlight %}

MobileInputMockLayout

MobileInputMockLayout is used to mock a mobile input but for use cases where a transition is necessary to be performed to another layout where an actual mobile input is presented to the user. This is primarily a mock layout that provides a callback for user click events that allows the developer to then perform any click events to other activities. As an example, two use cases that use this layout can be found in:

  • VendorLandingActivity in Vendor app
  • LandingActivity in User app

{% highlight xml %} {% endhighlight %}

Initialization

This component comes in-built with a country selection dropdown. An initialization from the code-level is necessary to provide the component with a Context. Therefore, in the onCreate of your activity, initialize this component as follows.

{% highlight java %} mobileInputMockLayout.initialize(this); {% endhighlight %}

Callbacks

{% highlight java %} mobileInputMockLayout.setMobileInputEventListener(new MobileInputMockLayout.MobileInputEventListener() { @Override public void onCountrySelected(CountryCodeRowItem item) { //Handle event where the user selects a country from the dropdown } @Override public void onMobileLayoutClicked() { //Handle event where the user taps on the mobile input mock layout } }); {% endhighlight %}

MobileInputLayout

MobileInputLayout provides a validated mobile input text field customized with a coutnry selection drop down. All mobile numbers are validated by any required country's ISO code. It also provides a callback when the number is changed by the user along with the validity of the entered number.

{% highlight xml %} {% endhighlight %}

Initialization

This component comes in-built with a country selection dropdown. An initialization from the code-level is necessary to provide the component with a Context. Therefore, in the onCreate of your activity, initialize this component as follows.

{% highlight java %} //list of country codes ArrayList countryCodes = new ArrayList<>(); String countryISOCode = "MY"; //ISO Alpha-2 Code mobileInputLayout.initialize(context, countryCodes, countryISOCode); {% endhighlight %}

Callbacks

{% highlight java %} mobileInputLayout.setMobileInputEventListener(new MobileInputLayout.MobileInputEventListener() { @Override public void onCountrySelected(CountryCodeRowItem item) { //handle event when the user selects a country } @Override public void onMobileValueChanged(String countryCode, String value, boolean isValidNumber) { //handle event when the user changes their input value //countryCode is the selected country code //value is the value entered by the user //isValidNumber is the mobile number validation for the entered value } }); {% endhighlight %} The validation is done by Google's libphonenumber library. The component is using a wrapper called android-phone-field. For ISO Alpha-2 codes, refer to [this guide](https://www.nationsonline.org/oneworld/country_code_list.htm).

ToastBanner Notification

The ToastBanner is an extended NotificationBanner singleton that was styled to match the Kaodim design guidelines to show success and error messages. It also allows to customize message text and duration and provide click listener to dismiss the banner. ToastBanner needs a layout view and activity as param to show the banner below the view, ex. toolbar and when activity is running on foreground.

Code level

Pre-Loader Animation

The PreLoaderAnimation is an extended NotificationBanner singleton that was styled to match the Kaodim design guidelines to render loading gif animation. This animation can be called anywhere anytime from your code. For best practice, create methods that invoke and dismiss the loader in base activity and call these methods in respective activities that extend baseactivity. Also, hide the animation when backpressed to avoid memory leakage.

Code level

Add this in base activity

{% highlight java %} public void showLoadingAnim(){ View view = findViewById(android.R.id.content); PreLoaderAnimation.showLoading(view,this); } public void hideLoadingAnim(){ PreLoaderAnimation.hideLoading(); } {% endhighlight %}

Add this in the respective activity to avoid memory leakage when user back press

{% highlight java %} @Override public void onBackPressed() { super.onBackPressed(); hideLoadingAnim(); } {% endhighlight %}

You can call the anim anywhere you want by calling showLoadingAnim() and hide the animation withhideLoadingAnim().
